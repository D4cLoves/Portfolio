## Как реализована форма

Форма — это три модуля, каждый со своей зоной ответственности:

1. **validator.ts** — чистые функции валидации. Ноль зависимостей от DOM. Проверяет имя, email, телефон, комментарий. Возвращает объект с ошибками.

2. **notification.ts** — управляет UI-состояниями. Показывает inline-ошибки под полями, переключает кнопку в loading/disabled, выводит success/error уведомления.

3. **form.ts** — оркестратор. Слушает submit, вызывает валидатор, если ок — шлёт POST на `/api/contact`, обрабатывает ответ.

На бэкенде — serverless-функция (`api/contact.ts`):
- Валидирует повторно (не доверяем клиенту)
- Отправляет 2 письма через Nodemailer: уведомление владельцу + копию пользователю
- Возвращает JSON с success/error

Также есть альтернативная реализация на **Symfony** в папке `backend/` — полноценный PHP-бэкенд с DTO, сервисами и Twig-шаблонами писем.

## AI-интеграция

На сайте есть чат-виджет (кнопка в правом нижнем углу). Посетитель может спросить что угодно обо мне — "Какой стек?", "Есть опыт с React?", "Какие проекты делал?" — и получить ответ от AI.

Под капотом:
- Serverless-функция `api/chat.ts`
- Groq API с моделью Llama 4 Scout
- System prompt с полным контекстом обо мне
- Обработка ошибок, лимит длины сообщения, typing-индикатор

## Что исправлял вручную

- Настройка SMTP для Mail.ru (пароли приложений, порты, SSL)
- Деплой на Vercel — env-переменные, редеплой после добавления ключей
- Выбор AI-провайдера — перебрал OpenAI → Gemini → Groq, пока не нашёл рабочий бесплатный вариант
- Подгонка стилей и адаптива под конкретный контент
- Финальная проверка доступности (aria-атрибуты, label/input связки, alt-тексты)
